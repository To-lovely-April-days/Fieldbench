## 1. 一句话定位

**给自动化/嵌入式工程师的现代化通信调试工作台:串口、Modbus 在同一条报文时间轴上,选中任何一帧,AI 告诉你它是什么、哪里错了、下一步查什么。**

长期形态是多协议插件化工作台(v1.x 加 TCP/UDP raw、MQTT,v2.0 加 CAN),v1.0 只交付最小可售卖切片。

## 4. 目标用户与核心场景

用户画像:自动化工程师(PLC/仪表/SCADA 集成商)、嵌入式固件工程师、设备厂商测试/售后。共同特征:Windows 笔记本 + USB 转 485,现场经常无网,公司可报销 $100 以内软件,极度厌恶订阅制工具和联网验证。

**核心使用循环**(功能存在性判据,不在循环上的功能不做):接入 → 看流 → 试探 → 诊断 → 修改 → 验证 → 留证。以下场景是该循环的具体展开,同时作为 UI 设计的走查清单。

- **S0 首次启动**:下载打开 → 直接弹"新建连接"(枚举到的 USB 串口默认选中)→ 60 秒内看到第一帧(TTFF < 60s)。手边没设备的下载者(约一半)走 Demo 模式,10 秒看到全部能力。
- **S1 现场联调**:接上 485 总线,想同时看到原始字节流和 Modbus 解析结果,快速判断是电气问题、参数问题还是协议问题。→ 统一时间轴 + 逐字节高亮。
- **S2 无设备开发**:上位机软件开发时手边没有 PLC,需要一个从站假装设备,还要能注入异常值。→ Slave 仿真 + 值发生器;自环测试(本机 master↔slave)天然需要 2 个连接 → Pro 转化钩子。
- **S3 看不懂的帧**:CRC 校验不过 / 收到异常码 / 满屏乱码,老工程师靠经验,新工程师靠搜索两小时。→ 选中帧 → AI 解释 → 原因排序 + 检查清单。
- **S4 记录取证**:设备偶发掉线,需要挂一晚上抓包,第二天导出给厂商。→ 大容量环形缓冲 + 时间戳 + 导出。

## 5. 核心概念与信息架构

### 5.1 三层抽象(平台化的根基,v1.0 必须按此实现)

```
Connection(物理/网络通道:串口、TCP)
  └─ Session(连接 × 协议视角:Raw / Modbus Master / Modbus Slave)
       └─ Frame Stream(统一报文流:时间戳、方向、原始字节、解析树)
```

- **字节流是事实源,帧只是视图**:存储层只存带时间戳、分方向的原始字节流;"帧"不落库,是当前 Lens 对字节流的派生解析结果。由此视角可回溯切换——用 Raw 观察数分钟后切到 Modbus,已捕获的全部历史重新切帧、着色、解析(Wireshark dissector 模型)。**此为 W1 抽象层验收项**;若把"帧"做成存储单位,切视角只对新数据生效,历史全废,后期无法补救。
- 会话三类:**监视(Monitor)**/ Master / Slave。Master/Slave 会话的协议由用户建会话时声明,不存在未知;监视会话缺省 Raw Lens、随时可切,协议未知不阻塞任何功能(时间轴的通用列——时间、方向、长度、hex——不依赖协议知识)。
- Raw 的切块边界仅为显示美化(Windows 串口读事件时序不精确),协议真边界由 Lens 以 CRC 滑窗等策略对字节流重算。
- 协议以内部接口(`IProtocolLens`:切帧、解析、着色、摘要)实现模块化。**v1.0 不做动态插件加载**,只保证接口干净,防止过度设计。
- 时间轴是所有协议共用的脊柱,后续新协议只需实现 Lens。

### 5.2 主窗口布局

```
┌──────────────────────────────────────────────────────┐
│ 顶栏:连接快捷操作 │ 会话标签页                        │
├────────┬─────────────────────────────┬───────────────┤
│ 左栏   │ 报文时间轴(虚拟化列表)      │ 右栏(可收起)  │
│ 连接/  │  时间│方向│长度│摘要│hex     │ ①字节检查器    │
│ 会话树 ├─────────────────────────────┤ ②AI 解释面板   │
│        │ 协议面板(随会话类型切换):   │               │
│        │ Raw→发送器 / Master→寄存器   │               │
│        │ 网格+轮询任务 / Slave→寄存器  │               │
├────────┴─────────────────────────────┴───────────────┤
│ 状态栏:TX/RX 字节、错误计数、帧率、连接时长、AI 余额   │
└──────────────────────────────────────────────────────┘
```

视觉基调:精密仪器风格,深浅双主题,等宽数字(tabular numerals),hex 用等宽字体,协议字段着色遵循固定色板(地址/功能码/数据/校验四色,全产品统一)。

**交互硬规则**(流程走查结论,UI 稿按此验收):

- 首跑不给空 dashboard,直接弹"新建连接";TTFF < 60s
- 连接参数热改:波特率等在工具栏直改、即时生效,会话与时间轴保留,禁止删连接重建
- AI 入口 = 异常帧行内图标,是试用→订阅的主转化位,不得移入菜单
- Slave 会话:寄存器网格与时间轴同屏(用户是两边对照看的)
- 曲线异常点点击 → 时间轴跳转至对应时刻报文

## 6. 功能需求

优先级:P0 = v1.0 发售阻塞项;P1 = 发售可缺、v1.0.x 两周内补齐。

### 6.1 连接管理(P0)

- 串口:端口热插拔枚举(显示友好名,如 `COM3 — CH340`)、波特率标准档 + 任意自定义值、数据位 5–8、校验 N/E/O/M/S、停止位 1/1.5/2、流控 None/RTS-CTS/XON-XOFF
- TCP Client(目标 IP:Port)、TCP Server(监听端口,多客户端接入)
- 连接配置档:命名保存、最近列表、一键重连;可选断线自动重连(间隔可配)
- 参数热改:已连接状态下修改波特率/校验等即时生效,会话与已捕获数据保留
- Free 限 1 个活动连接,Pro 无限

**验收**:USB 转串口插拔后列表 1s 内刷新;设备异常拔出不崩溃,会话保留可重连;CH340 / FTDI / CP210x / PL2303 实测通过。

### 6.2 报文时间轴(P0,平台脊柱)

- 列:时间戳(ms 精度,绝对/相对可切)、方向(TX/RX 图标+色)、长度、协议摘要、原始 hex
- 切帧策略:
  - Raw 串口:静默间隔(默认 20ms 可调)/ 定长 / 结束符
  - Modbus RTU:**3.5T 静默 + CRC 滑窗回溯混合**(高波特率下 Windows 定时精度不可靠,纯 3.5T 会粘帧,必须混合策略)
  - Modbus TCP:按 MBAP length 字段
- 字节检查器:选中帧 → 字节网格,按协议字段着色;悬停任意字节 → tooltip 显示字段名、十进制/十六进制/二进制值
- 过滤:方向、hex/ASCII 关键字、功能码、仅异常帧
- 环形缓冲默认 100,000 帧(可配);暂停渲染不暂停采集
- **协议自动识别**:监视/Raw 流上运行启发式——连续 ≥3 块 CRC16-Modbus 校验通过(单帧撞对概率 1/65536,三帧即确定)或 MBAP 头结构一致 → 时间轴顶部非模态 chip:"检测到 Modbus RTU · 12/12 帧 CRC 通过 — 切换解析?"一键切换,历史全量重解析。识别不出的流可选中数块交给 AI 猜测协议(§6.7 顺带覆盖,是未来协议逆向功能的入口)
- **总线被动监听方向推断**:挂在 485 总线上被动抓包时物理方向全为 RX,Modbus Lens 按帧语义(请求/响应结构)推断角色,时间轴显示 M→S / S→M(Modbus Poll 无总线监听能力,差异化点)
- 写盘记录模式(P1,v1.0.x):record-to-disk 边采边写文件,环形缓冲只负责屏幕显示;挂机取证刚需(500ms 轮询 12h ≈ 17 万帧,超出缓冲上限,不写盘则早晨异常帧已被冲掉)
- 导出:选中/全部 → CSV、JSON、原始 bin、txt 日志

**验收**:115200 波特率满速连续流 30 分钟,UI ≥ 30fps、零丢帧;10 万帧驻留内存 < 300MB;纯 Modbus RTU 流 ≤ 5 帧内触发识别 chip,随机字节流零误报;Lens 回溯切换后历史帧解析结果与实时解析一致。

### 6.3 原始终端 / 发送器(P0)

- hex 编辑(空格分隔自动格式化)/ASCII 切换,支持 `\r` `\n` `\xNN` 转义
- 校验自动追加:无 / CRC16-Modbus / CRC16-CCITT / XOR / SUM
- 定时循环发送(间隔 ms、次数/无限);发送历史;常用帧命名收藏
- 快捷键:Enter 发送(可配)

### 6.4 Modbus Master(P0)

- RTU / TCP;TCP 校验事务 ID,支持请求排队
- FC:01/02/03/04/05/06/0F/10;高级模式支持自定义原始 PDU
- 轮询任务:每任务 = 从站地址 + FC + 起始地址 + 数量 + 周期(ms);多任务并行(同连接内串行调度);启停/单发
- 寄存器网格:地址、原始值、解析值、用户标签(名称备注);单元格直接编辑 → 自动选择 05/06/0F/10 下发(可手动指定)
- 数据类型:bit / int16 / uint16 / int32 / uint32 / float32 / double / string;字节序四种(AB CD / CD AB / BA DC / DC BA),每标签独立配置
- 异常码翻译成人话(如 `02 Illegal Data Address — 从站不存在该地址,核对起始地址与数量是否越界`)
- 超时/重试每连接可配,默认 1000ms × 3
- **从站扫描**:遍历从站地址 1–247(探测 FC 可选,短超时默认 200ms),列出响应者;可选叠加常见串口参数矩阵(波特率 × 校验)扫描。超时排障的第一动作,ModbusSimulator 已有 address scanner,必须对齐

**验收**:与 Modbus Poll、真实 PLC(至少两个品牌)互测全功能码通过;8 种异常码全部正确翻译。

### 6.5 Modbus Slave 仿真(P0,Pro 专属)

- RTU(串口)/ TCP(监听,多 master 接入);从站 ID 可配
- 四类寄存器区(0x 线圈 / 1x 离散输入 / 3x 输入寄存器 / 4x 保持寄存器),65536 地址空间稀疏存储
- 值编辑:网格直改;值发生器:固定 / 递增 / 随机范围 / 正弦(幅值周期可配)——演示与无设备联调的核心卖点
- 请求日志进同一时间轴;非法地址/功能码自动回异常
- P1:错误注入(指定概率不响应 / 延时 / 回错误码)——竞品 ModbusSimulator 有此功能,尽快跟上

**验收**:本机自环(master 会话 ↔ slave 会话,经 com0com 虚拟串口对或 TCP 127.0.0.1)全功能码通过。

### 6.6 数据曲线与导出(P1)

- 从轮询任务标签拖入图表;多通道自动配色;滚动窗口 60s/10min/全部;暂停/缩放
- **时间轴联动**:曲线异常点点击 → 时间轴跳转至该时刻报文 → 可直接 AI 解释。"值不对 → 是哪帧 → 为什么"这条链全品类独有,实现仅为时间戳查找
- W3 内优先级排在 AI 点表导入之后;溢出则按既定规则整体移 v1.0.x,不拖发售
- 导出 CSV(时间戳 + 各通道)
- Free 限 2 通道,Pro 无限
- 图表库选型:ScottPlot(渲染性能优先)

### 6.7 AI 报文解释(P0,差异化功能,全品类竞品均无)

- **入口**:时间轴选中 1–N 帧 → 右键"AI 解释"/右栏按钮;异常帧行内快捷图标
- **上下文打包**:协议类型、连接参数(波特率等)、选中帧(hex + 已解析字段树)、前后 N 帧摘要(默认 20 可配)、用户补充问题(可选输入框)
- **输出**(流式渲染):这是什么帧 → 字段逐项含义 → 异常点 → 最可能原因(按概率排序)→ 下一步检查清单
- **必须打穿的 5 个标杆场景**(发售前人工评测 ≥ 4/5 分):
  1. CRC 校验失败(区分波特率不匹配 / 线路干扰 / 字节序颠倒的特征)
  2. 异常码 01/02/03/04 各自的排查方向
  3. 满屏乱码(识别波特率错误的典型字节特征)
  4. 请求发出无响应(结合连接参数生成核对单:地址、终端电阻、AB 线反接…)
  5. 正常帧(准确讲解每个字段,建立信任)
- **配额**:所有用户送 30 次试用;订阅 1000 次/月合理使用;本地 + 网关双侧计数
- **网关**:Cloudflare Workers 薄代理(推荐,零运维;备选 .NET 部署海外 VPS),license key 鉴权,**不落盘存储报文内容**;模型用低成本快速档,目标单次成本 < $0.01(订阅毛利 > 85%)
- **隐私**:首次使用弹窗说明发送内容范围;设置页提供"将发送什么"预览
- **断网**:按钮置灰,tooltip 注明"AI 需联网,核心功能不受影响"

**验收**:5 场景评测达标;P95 端到端延迟 < 6s;断网时其余功能零影响。

### 6.8 AI 点表导入(P0,发售叙事主角)

工程师拿到新设备的第一个小时都花在抄手册点表上;此功能把一小时压成 60 秒,是演示视频的开场。

- **输入**:剪贴板截图粘贴(手册寄存器表截图,vision 提取)或粘贴表格文本;PDF 整本上传 + 页码选择放 v1.0.x
- **提取**:AI 输出结构化点表 JSON——名称、寄存器区(0x/1x/3x/4x)、地址、数据类型、字节序、缩放/偏移、单位、读写属性、备注
- **地址基准强制确认**:入库前用户必须二选一(40001 型 PLC 地址 / 0 基协议地址)。这是点表导入出错的头号来源,不允许 AI 猜
- **审核界面是灵魂**:AI 结果全部以"待确认"态呈现,越界/地址重叠/类型冲突自动标红,可改可删,确认后才入库;绝不静默导入
- **落地即可测**:确认后写入寄存器网格标签,自动生成分段轮询任务(连续地址合并,遵守单次 125 寄存器上限)→ "一键全读"验证通信,缩放与单位即时生效
- **设备档案**:导入结果可保存为设备档案文件(JSON)复用与分享——后续设备档案库的资产起点
- 计费:订阅含 30 次/月,免费送 3 次(vision 调用成本约为文本解释十倍,必须独立配额);隐私告知同 §6.7

**验收**:3 份真实设备手册(截图输入)提取,经审核界面修正后字段准确率 ≥ 95%;强制基准确认使地址基准误判归零;P95 提取延迟 < 20s(流式进度)。

- license = Ed25519 签名文件,绑定机器指纹(CPU + 主板 + 系统盘序列号哈希,容忍单项变更);1 license = 2 台设备,官网自助解绑
- Pro 功能 14 天全功能试用,无需信用卡
- **激活后永久离线可用,无任何回连校验**(工程师红线,官网明示)

## 9. 技术方案要点

- **栈**:.NET 8 + Avalonia 11(复用 ParallelReactor 工程的控件与主题资产);串口 `System.IO.Ports` + 必要时 Win32 覆写;Modbus 协议栈自研(已有工具箱代码,不引第三方库,为后续 Lens 接口铺路)
- **时间轴渲染**:虚拟化列表 + 环形缓冲,解析结果懒计算缓存;这是全产品性能核心,W1 先立性能测试脚手架
- **AI 网关**:Cloudflare Workers,职责仅三件:license 鉴权、配额计数、转发模型 API;不落库
- **license 工具链**:Ed25519 签名器(离线保管私钥)+ Paddle webhook → 发 key;离线激活自助页挂官网静态站 + 一个 serverless 函数
